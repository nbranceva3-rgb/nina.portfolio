Usability Research

Below are the Customer Journey Maps compiled from usability testing sessions conducted after the initial design phase. Three participants were tested — Woman, 23, Woman, 28, and Man, 29 — representing the core target audience of students and working adults who manage heavy browser tab usage. Findings from these sessions shaped the affinity diagram and directly informed two key design iterations.

Participant 01

Woman, 23 — Student

Stage Actions Thoughts Emotion Pain points Opportunities
Install & launch Installs the extension, opens the popup for the first time "Looks clean, let's see what it does" Curious Unsure what to do first — no visible prompt Add a short welcome message or tooltip on first open
Onboarding — import Sees open tabs, clicks Import All "Oh, it grabbed my tabs automatically — nice" Pleased Not sure if she can undo the import Add a subtle "undo" note after import
Creating a folder Tries to name a folder, uses text button to confirm "Where's the Save button? I don't see it" Confused Text button for saving action is too subtle — missed it entirely Make action buttons visually prominent (→ Iteration 01)
Moving tabs to folder Drags tab into folder successfully after finding the action "This actually makes sense, like a bookmark but smarter" Satisfied — Reinforce this mental model in onboarding copy
Encountering Focus Mode Taps the Focus Mode toggle without reading what it does "What did that just do? My tabs are gone?" Alarmed No explanation of what Focus Mode is before activation Add explanation screen before paywall (→ Iteration 02)
Paywall Sees upgrade prompt, pauses, closes without upgrading "I'd consider it if I understood what I'm actually getting" Hesitant Value unclear at decision moment Show concrete benefits on the paywall screen

Participant 02

Woman, 28 — Working adult

Stage Actions Thoughts Emotion Pain points Opportunities
Install & launch Installs, opens popup; already has 40+ tabs open "Finally something for this mess" Hopeful Slightly overwhelmed by the number of tabs listed Consider grouping or collapsing large tab lists by default
Onboarding — import Selects specific tabs to import rather than all "I only want the work ones, not all the shopping tabs" In control — Selective import flow already works well — keep it
Creating a folder Creates folder "Work", tries to confirm — scans the screen twice "There must be a Save somewhere…" Frustrated Same issue: text button for confirmation is invisible in context Increase visual weight of action buttons (→ Iteration 01)
Moving tabs to folder Successfully organises 12 tabs into 3 folders "This is actually faster than I expected" Impressed — Could highlight this speed advantage in marketing
Encountering Focus Mode Reads tooltip, taps Focus Mode intentionally "OK so it hides everything else — makes sense for deep work" Confident Still wants a confirmation before it activates Add a one-time confirmation or brief animation showing what changes
Paywall Reads the upgrade screen carefully; considers upgrading "The price is fine but I need to know if this actually works for me" Evaluating No trial option shown Mention free trial or money-back option on paywall

Participant 03

Man, 29 — Working adult

Stage Actions Thoughts Emotion Pain points Opportunities
Install & launch Installs extension quickly; skims the interface without reading "How hard can this be" Confident Skips instructions; pattern-matches to similar tools Design for exploratory users — make the interface self-evident
Onboarding — import Clicks Import All without hesitation "I'll sort them out afterwards" Pragmatic — Bulk import works well for this user type
Creating a folder Types folder name and waits — nothing happens; tries clicking around "Hmm, where do I confirm this?" Confused Confirmation action is a text label with no button affordance — not obvious Text buttons must look like buttons (→ Iteration 01)
Moving tabs to folder Moves tabs using drag-and-drop; discovers context menu by accident "Oh, right-click works too — that's handy" Surprised positively Context menu discoverability is low Surface the right-click option more visibly in onboarding
Encountering Focus Mode Activates Focus Mode and immediately looks for a way to undo it "What happened to all my tabs? Is this reversible?" Anxious No clear explanation of what Focus Mode does before it activates; feels irreversible Explain Focus Mode before paywall, show it's reversible (→ Iteration 02)
Paywall Sees paywall; reads quickly; decides to stay on Free tier for now "I'll upgrade if I keep using it — not yet" Neutral Doesn't feel the urgency to upgrade right now Consider delayed upgrade prompts after proven value is established
